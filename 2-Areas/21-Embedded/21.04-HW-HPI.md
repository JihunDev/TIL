# HW - HPI (Host Port Interface)

## 개요
- **핵심 개념**: Host Processor와 DSP(C54x) 간 병렬 데이터 통신 인터페이스
- **특징**: DSP는 Slave로만 동작 — Host가 DSP 내/외부 메모리에 직접 액세스 가능
- **관련 기술 비교**: SPI (직렬) vs HPI (병렬)

## 상세 분석

### HPI 레지스터 3종

| 레지스터 | 이름 | 액세스 주체 | 역할 |
|---------|------|------------|------|
| HPIA | HPI Address Register | Host 전용 | Host가 접근할 HPI RAM 주소 설정 |
| HPIC | HPI Control Register | Host + C54x 쌍방 | 제어 및 Status bit 관리 |
| HPID | HPI Data Register | Host 전용 | 실제 데이터 read/write 경로 |

### Host → HPI 액세스 순서

```
1. HPIC 설정 (Control register 초기화)
2. HPIA 설정 (접근할 RAM 주소 지정)
3. HPID 경유 데이터 read / write
```

### ⚠️ HPID 액세스 핵심 주의사항

- HPI RAM 및 레지스터는 **16비트** 단위
- HPI 버스 자체는 **8비트** → 모든 액세스는 **상위/하위 2바이트로 분할** 전송 필요
- **반드시 2바이트 연속 액세스** — 상위만 또는 하위만 단독 액세스 불가

```
16-bit data
┌──────────┬──────────┐
│ High Byte│ Low Byte │
│ (1st)    │ (2nd)    │
└──────────┴──────────┘
  ← 반드시 연속 전송 →
```

## 엣지 케이스 / 잠재적 버그
- [ ] 2바이트 비연속 액세스 시 데이터 오염 — 인터럽트로 중간에 끊기지 않도록 atomic 처리 필요
- [ ] HPIC 초기 설정 누락 시 HPI 동작 불안정

## 실무 적용 포인트
- [ ] Host MCU에서 HPI 드라이버 작성 시 2바이트 전송 함수를 원자적으로(atomically) 구현
- [ ] DMA를 활용하면 CPU 개입 없이 HPI 데이터 전송 가능 → 처리량 향상
