# HW - SPI (Serial Peripheral Interface Bus)

## 개요
- **핵심 개념**: Master-Slave 모드 기반의 4-wire 동기식 직렬 통신 프로토콜
- **특징**: 전이중(Full-Duplex) 통신, 단일 Master, 다수 Slave 지원
- **관련 기술 비교**: I²C (2-wire, 주소 방식) vs SPI (4-wire, CS 선택 방식)

## 상세 분석

### 신호 핀 정의

| Signal Name | Alternative Names | 방향 | 설명 |
|-------------|-------------------|------|------|
| SCLK | SCK, CLK | Master → Slave | Serial Clock (동기화 신호) |
| MOSI | SDI, DI, SI, SIMO | Master → Slave | Master Out Slave In |
| MISO | SDO, DO, SO, SOMI | Slave → Master | Slave Out Master In |
| SS | nCS, CS, nSS, STE, CE | Master → Slave | Slave Select (Active Low) |

### 동작 구조

- Circular Queue(링 버퍼)와 유사한 시프트 레지스터 방식
- SCLK 엣지에 맞춰 MOSI/MISO 비트 교환 → 전이중 통신

```
Master                    Slave
┌─────────────────────────────┐
│  Shift Register (8-bit)     │
│  MOSI ──────────────────>   │
│  MISO <──────────────────   │
│  SCLK ──────────────────>   │
│  SS   ──────────────────>   │
└─────────────────────────────┘
```

### SPI 모드 번호 (CPOL / CPHA)

| 모드 | CPOL | CPHA | 클럭 휴지 상태 | 데이터 샘플 타이밍 |
|------|------|------|--------------|-----------------|
| 0 | 0 | 0 | Low | 상승 엣지 |
| 1 | 0 | 1 | Low | 하강 엣지 |
| 2 | 1 | 0 | High | 하강 엣지 |
| 3 | 1 | 1 | High | 상승 엣지 |

> 가장 많이 사용: **Mode 0** (CPOL=0, CPHA=0)

### 장단점

| 장점 | 단점 |
|------|------|
| 완전한 전이중(Full-Duplex) 통신 | 하드웨어 Slave 인식 없음 |
| 단순한 하드웨어 (트랜시버 불필요) | 하드웨어 흐름 제어 없음 |
| 4핀만으로 구성 | 오류 검사 프로토콜 미정의 |
| 빠른 통신 속도 | 노이즈 스파이크에 취약 |
| 프로토콜 유연성 | 단일 Master만 지원 |
| | 칩간 근거리 통신에만 적합 (RS-485보다 거리 짧음) |

## 엣지 케이스 / 잠재적 버그
- [ ] SS 핀을 소프트웨어로 제어 시 타이밍 문제로 Slave 오인식 가능
- [ ] SPI 모드(CPOL/CPHA) 불일치 → 데이터 오염, 디버깅이 어려움
- [ ] 여러 Slave 연결 시 비활성 Slave의 SS를 High로 유지하지 않으면 버스 충돌

## 실무 적용 포인트
- [ ] IMU, 가속도계 등 센서 연결 시 데이터시트에서 SPI 모드 반드시 확인
- [ ] DMA + SPI 조합으로 CPU 부하 없이 고속 데이터 전송 가능
- [ ] CAN 버스와 비교: CAN은 장거리/노이즈 환경, SPI는 보드 내 칩간 고속 통신에 적합
