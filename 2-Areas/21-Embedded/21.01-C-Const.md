# C - `const` / Pointer Const

## 개요
- **핵심 개념**: `const`로 선언된 변수는 읽기 전용으로, 선언 시 초기화 후 값 변경 불가
- **관련 개념**: pointer const — 포인터 자체와 가리키는 값 중 무엇을 상수로 만드느냐에 따라 동작 차이

## 상세 분석

### `const` 기본 사용

```c
const (자료형) (상수명) = (상수 값);
```

- 선언 시 값을 반드시 지정해야 함
- 변경 시도 시 컴파일 에러: `Assignment of read-only variable '상수명'`

### Pointer Const — 두 가지 패턴

#### 패턴 1: 값을 상수로 (포인터는 이동 가능)

```c
int a = 10;
int b = 20;
const int* c = &a;

*c = 100; // ❌ 에러: 가리키는 값 변경 불가
c = &b;   // ✅ OK: 포인터가 가리키는 대상은 변경 가능
```

- `const int*` : c가 가리키는 **값**이 상수

#### 패턴 2: 포인터를 상수로 (값은 변경 가능)

```c
int a = 10;
int b = 20;
int* const c = &a;

*c = 100; // ✅ OK: 가리키는 값 변경 가능
c = &b;   // ❌ 에러: 포인터 자체를 변경 불가
```

- `int* const` : c 포인터 자신이 상수

### 비교 요약

| 선언 | 값 변경 | 포인터 이동 |
|------|---------|------------|
| `const int* p` | ❌ | ✅ |
| `int* const p` | ✅ | ❌ |
| `const int* const p` | ❌ | ❌ |

## 엣지 케이스 / 잠재적 버그
- [ ] `const`를 캐스팅으로 우회(`const_cast`)하면 undefined behavior 발생 가능
- [ ] 함수 파라미터에서 `const char*` vs `char* const` 혼동 주의

## 실무 적용 포인트
- [ ] 임베디드: 읽기 전용 레지스터 주소를 `const volatile*`로 선언하여 컴파일러 최적화 방지
- [ ] API 설계 시 입력 파라미터는 `const T*`로 선언하여 의도 명확화
