# Signal - DAQ (Data Acquisition System)

## 개요
- **핵심 개념**: 전압, 전류, 온도, 압력, 소음 등 물리적 현상을 디지털 데이터로 수집하는 시스템
- **시스템 구성**: `Sensor → DAQ Device → Computer`

## 상세 분석

### 시스템 구성 흐름

```
물리 현상 → [Sensor] → 전기 신호 → [Signal Conditioning] → [ADC] → [Computer Bus] → [Software]
```

### 1. Sensor (트랜스듀서)

물리적 현상을 전기 신호로 변환. 센서 유형에 따라 출력이 전압/전류/저항/시간 등으로 다름.

| 센서 | 측정 현상 |
|------|----------|
| 열전쌍, RTD, 써미스터 | 온도 |
| 광센서 | 빛 |
| 마이크 | 소음 |
| 스트레인 게이지, 압전 센서 | 힘, 압력 |
| 전위차계, LVDT, 광학 엔코더 | 위치, 변위 |
| 가속도계 | 가속도 |
| pH 전극 | pH |

### 2. DAQ Device

컴퓨터와 외부 신호 사이의 인터페이스. 아래 3요소로 구성:

#### Signal Conditioning (신호 조절 회로)
- 원본 신호는 노이즈가 많거나 직접 측정 위험할 수 있음
- ADC 입력에 적합한 형태로 신호 가공 (증폭, 필터링, 절연)

#### ADC (Analog-to-Digital Converter)
- 아날로그 신호를 디지털로 변환
- 미리 정의된 샘플링 속도로 주기적으로 신호 샘플링
- 핵심 파라미터: **분해능(Resolution)**, **샘플링 속도(Sample Rate)**

#### 컴퓨터 버스 인터페이스
- USB, PCI, PCI Express, 이더넷 등

#### 추가 기능
- DAC: 디지털 → 아날로그 출력 (제어 신호 출력)
- Digital I/O: 디지털 입출력
- 카운터/타이머: 디지털 펄스 카운팅 및 생성

### 3. Computer
- DAQ 동작 제어
- 측정 데이터 처리, 시각화, 저장
- Driver Software + Application Software (예: LabVIEW, Python NI-DAQmx)

## 엣지 케이스 / 잠재적 버그
- [ ] 샘플링 속도가 나이퀴스트 조건 미충족 시 에일리어싱(Aliasing) 발생 → 안티 에일리어싱 필터 필요
- [ ] 신호 컨디셔닝 없이 고전압 신호 직접 연결 시 DAQ 장비 손상 가능

## 실무 적용 포인트
