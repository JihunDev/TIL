# Vision - 영상 표현 방식: Binary / Grayscale / Color

## 개요
- **핵심 개념**: 디지털 영상은 픽셀 값의 비트 수와 채널 수에 따라 Binary, Grayscale, Color로 분류
- **관련 기술 비교**: 메모리 사용량 및 처리 속도 측면에서 Binary < Grayscale < Color

## 상세 분석

### 영상 종류 비교

| 종류 | 비트 수 | 픽셀 값 범위 | 특징 |
|------|---------|------------|------|
| Binary | 1 bit | 0 (흑), 1 (백) | 최소 메모리, 윤곽 검출 |
| Grayscale | 8 bit (일반) | 0 ~ 255 | 명도 정보만 |
| Grayscale | 16 bit | 0 ~ 65535 | 의료/과학 영상 |
| Color (RGB) | 24 bit (3채널) | [0~255]³ | 가산혼합 |
| Color (CMYK) | 32 bit (4채널) | [0~255]⁴ | 인쇄용, 감산혼합 |

### Color 모델 상세

#### RGB
- 빨강(R), 초록(G), 파랑(B) 3가지 빛의 **가산혼합(Additive)**
- 모든 채널 최대 → 흰색 / 최소 → 검정

#### HSI / HSV
- Hue(색상), Saturation(채도), Intensity/Value(명도)
- 인간의 색 인식 방식에 가까운 모델
- **색상/색조 (Hue)**: 해당 색의 원색 (0°~360°)
- **채도 (Saturation)**: 색의 순수도 (0 = 회색, 1 = 순수한 색)
- **명도 (Intensity)**: 밝기

#### CMY / CMYK
- 시안(Cyan), 마젠타(Magenta), 노랑(Yellow)의 **감산혼합(Subtractive)**
- 인쇄용, 모든 채널 최대 → 검정 (+K 채널로 순수 검정 표현)

## 실무 적용 포인트
- [ ] 이진화(Binarization) 전처리: 임계값(Threshold) 선택이 성능에 결정적
- [ ] 색상 기반 객체 검출 시 RGB 대신 **HSV** 사용 — 조명 변화에 더 강인함
- [ ] 그레이스케일 변환 시 권장 공식: `Y = 0.299R + 0.587G + 0.114B` (ITU-R BT.601)
