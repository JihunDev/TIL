# Git - 서로 다른 저장소 병합

## 핵심 요점
> 단순 폴더 복사는 커밋 이력이 사라진다. `--allow-unrelated-histories` 옵션으로 이력을 보존한 채 병합해야 한다.

## 내용

### 배경
- 두 저장소를 하나로 통합해야 할 때 (예: 서브 프로젝트 → 모노레포 통합)
- 폴더 복사 방식: 파일은 옮겨지지만 커밋 이력이 소실됨

### 저장소 병합 절차

```bash
# 1. 병합할 원격 저장소를 remote로 추가
git remote add project2 ../project2

# 2. 해당 저장소의 커밋 이력 가져오기
git fetch project2

# 3. 서로 관련 없는 이력 병합 허용 옵션으로 merge
git merge --allow-unrelated-histories project2/master

# 4. 임시 remote 제거
git remote remove project2

# 5. 병합 커밋 생성
git commit -m 'Merge project2 into project1'
git push
```

### `.gitignore` 충돌 발생 시

```bash
git reset .gitignore
```

### 주의사항 / 함정
- 두 저장소에 동일한 파일명이 있으면 merge conflict 발생 → 수동 해결 필요
- `--allow-unrelated-histories` 없이 merge 시 `fatal: refusing to merge unrelated histories` 에러

## 실무 적용 포인트
- [ ] 프로젝트 구조 개편 시 하위 모듈 이력을 보존한 채 메인 저장소로 통합 가능
