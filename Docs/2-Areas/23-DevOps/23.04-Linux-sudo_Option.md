# Linux - `sudo -S` / `-kS` 옵션

## 핵심 요점
> `sudo -S`는 표준 입력(stdin)으로 비밀번호를 받는 옵션. 스크립트 자동화에 사용되지만 **히스토리에 비밀번호가 노출**되는 심각한 보안 취약점이 있다.

## 내용

### 배경
- 스크립트나 파이프라인에서 비밀번호 입력 프롬프트 없이 sudo를 실행해야 할 때 사용

### `-S` 옵션: stdin에서 비밀번호 읽기

```bash
echo '비밀번호' | sudo -S 명령어
# 예시
echo '1234' | sudo -S vi test.txt
```

- `sudo`가 키보드 대신 **표준 입력(stdin)**에서 비밀번호를 읽음

### `-kS` 옵션: 토큰 리셋 후 stdin에서 읽기

```bash
echo '비밀번호' | sudo -kS 명령어
# 예시
echo '1234' | sudo -kS vi test.txt
```

- `-k`: 기존 sudo 액세스 토큰을 **강제 리셋**
- 이미 sudo가 활성화된 상태에서 `-S`만 사용하면 토큰 충돌로 오류 발생 → `-kS`로 해결

### ⚠️ 보안 주의사항

| 위험 | 설명 |
|------|------|
| 히스토리 노출 | `~/.bash_history`에 비밀번호가 평문으로 저장됨 |
| 보안 취약 | 관리자 권한 획득 방법이 스크립트에 하드코딩 |

**권장 대안**:
- `NOPASSWD` sudoers 설정으로 특정 명령어만 비밀번호 없이 허용
- SSH 키 기반 인증 + sudoers 조합

## 실무 적용 포인트
- [ ] CI/CD 파이프라인에서 sudo 필요 시 secrets 관리 도구 사용 (GitHub Secrets, Vault 등)
- [ ] 임시 스크립트에서 사용 후 히스토리 정리: `history -c` 또는 `HISTIGNORE` 설정
