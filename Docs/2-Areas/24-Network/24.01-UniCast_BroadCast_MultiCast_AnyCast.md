# Network - UniCast / BroadCast / MultiCast / AnyCast

## 개요
- **핵심 개념**: 네트워크에서 데이터 패킷의 전송 대상 범위에 따른 통신 방식 분류
- **관련 기술 비교**: 수신자 수와 선택 방식에서 차이

## 상세 분석

### 한눈에 비교

| 방식 | 관계 | 수신자 선택 | 라우터 통과 | 주요 사용 예 |
|------|------|------------|------------|-------------|
| UniCast | 1:1 | 특정 MAC/IP | ✅ | 일반 TCP/IP 통신 |
| BroadCast | 1:전체 | LAN 전체 | ❌ | ARP, DHCP |
| MultiCast | 1:그룹 | 수신 희망 그룹 | △ (멀티캐스트 라우터 필요) | 스트리밍, IPTV |
| AnyCast | 1:최근접 1 | 그룹 중 최적 노드 | ✅ | DNS 루트 서버, CDN |

### UniCast
- MAC Address 기반 1:1 통신 — 현재 가장 범용적으로 사용
- 수신자가 아닌 장비는 랜카드에서 Frame을 버림 → 다른 PC의 CPU 부하 없음
- **단점**: 동일 데이터를 다수에게 보낼 때 패킷을 각각 N번 전송 → 네트워크 효율 저하

### BroadCast
- Local LAN 내 모든 장비에게 1:다 전송
- BroadCast 주소로 오면 자신의 MAC이 아니어도 CPU까지 전달 → **모든 장비에 인터럽트 발생**
- 라우터를 통과하지 못함 (동일 네트워크 세그먼트 내에서만)
- 수신 장비들에게 거부권 없음

### MultiCast
- 수신 희망 그룹에게만 전송 (그룹 등록 필요)
- 스위치/라우터의 MultiCast 지원 필요
- 일반 라우터가 미지원 시 → **터널링 + 캡슐화**로 유니캐스트처럼 전송

### AnyCast
- 동일 IP를 가진 그룹 내에서 **가장 가깝거나 최적인 노드** 한 곳에만 전달
- 예: DNS 루트 서버(전 세계에 같은 IP로 배포), CDN 엣지 서버

## 엣지 케이스 / 잠재적 버그
- [ ] BroadCast 과다 발생 시 네트워크 폭주 (Broadcast Storm) → VLAN으로 도메인 분리
- [ ] MultiCast IGMP snooping 미설정 시 BroadCast처럼 모든 포트로 전달될 수 있음

## 실무 적용 포인트
- [ ] ROS2 DDS 통신: MultiCast 기반 → 멀티캐스트 라우팅 미지원 환경에서 Unicast 모드로 전환 필요
- [ ] 임베디드 CAN 버스: BroadCast 방식 → 수신 필터링을 CAN ID로 하드웨어 처리
