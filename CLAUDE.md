## Project Overview

This is a personal TIL (Today I Learned) knowledge base — a static documentation site using [Docsify](https://docsify.js.org/), deployed to both **GitHub Pages** and **GitBook**. There is no build pipeline or compilation step; all content is plain Markdown rendered client-side by Docsify. Sidebars (`_sidebar.md` for Docsify, `SUMMARY.md` for GitBook) are auto-generated by `generate-sidebar.py` and kept in sync via GitHub Actions.

## Local Development

```bash
npm i docsify-cli -g   # one-time install
docsify serve ./Docs   # serve at http://localhost:3000
```

## Content Creation

Use the automation script for new entries:

```bash
# One-time setup
chmod +x Docs/scripts/create-til.sh
echo 'alias til="/path/to/TIL/Docs/scripts/create-til.sh"' >> ~/.bashrc

# Usage
til quick "topic"       # quick note (80% of use cases)
til tech  "topic"       # technical analysis
til bug   "topic"       # troubleshooting record
til cheatsheet "topic"  # command/tool reference
til code  "topic"       # code analysis
til exp   "topic"       # experiment/benchmark
```

Templates live in `Docs/3-Resources/31-Templates/`.

### File Naming Convention

```
[CategoryNumber].[SequenceNumber]-[Topic].md
[CategoryNumber].[SequenceNumber]-Bug-[Topic].md   # troubleshooting
[CategoryNumber].[SequenceNumber]-Exp-[Topic].md   # experiment/benchmark
```

Example: `23.04-Linux-sudo_Option.md`, `21.02-Bug-CAN_Timeout_Error.md`

### Frontmatter

## Navigation (Auto-generated)

**`Docs/_sidebar.md` and `Docs/SUMMARY.md` are auto-generated — do NOT edit manually.**

They are regenerated automatically:
- **Locally**: `create-til.sh` calls `generate-sidebar.py` after every new file
- **CI**: GitHub Actions regenerates them on every push to `master`

To regenerate manually:
```bash
python3 Docs/scripts/generate-sidebar.py
```

Title extraction priority per document: YAML `title` field → first H1 → filename.

## Deployment

| Platform | URL | Trigger |
|---|---|---|
| **GitHub Pages** | `https://jihundev.github.io/TIL/` | Push to `master` → `gh-pages` branch |
| **GitBook** | gitbook.com (manual sync setup) | Push to `master` (after GitHub sync enabled) |
| Netlify (legacy) | https://jihun2til.netlify.app/ | Can be disabled |

### GitHub Pages Setup (one-time)
1. GitHub repo → Settings → Pages → Source: **Deploy from branch** → `gh-pages` / `/ (root)`
2. Push to `master` — GitHub Actions deploys automatically

### GitBook Setup (one-time)
1. [gitbook.com](https://gitbook.com) → New Space → Import from GitHub
2. Select `JihunDev/TIL` repo, branch `master`
3. GitBook reads `.gitbook.yaml` → uses `Docs/` as root, `SUMMARY.md` as TOC

### Sidebar Auto-generation Flow
```
til quick "topic"
  └─ create-til.sh
       ├─ creates .md file
       └─ generate-sidebar.py
            ├─ Docs/_sidebar.md  (Docsify / GitHub Pages)
            └─ Docs/SUMMARY.md   (GitBook)
git push master
  └─ GitHub Actions (.github/workflows/deploy.yml)
       ├─ runs generate-sidebar.py (re-sync)
       ├─ commits updated sidebars [skip ci]
       └─ deploys Docs/ → gh-pages branch
```
